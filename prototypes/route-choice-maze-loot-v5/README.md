# 古墓不让进 — 地图探索规则 V5 副本

本目录由 `route-choice-maze-loot-v4` 独立复制，在保留背包、满包拾取、局外养成和收藏系统的基础上增加章节地图探索规则。

反引号键（`Backquote`，键盘上的 `·` 键）打开开发者地图模式弹窗，可实时切换“探索规则”和“旧版全解锁”；弹窗打开时游戏暂停，不恢复任意节点跳转。

## 本副本改动

- 首次进入尚未通关的章节时启用战争迷雾，只显示当前/已探索房间及其相邻房间，探索后逐步揭开地图。
- 章节首次通关后再次进入时默认显示完整地形；BOSS 房和撤离点始终显示类型，其他未探索房间只在成为当前可选方向后揭示类型。
- 本局地图规则以进入时的章节通关记录为准，局内首次击败 BOSS 不会让当前地图突然切换显示方式。
- `·` 键开发者弹窗提供新旧地图规则实时对比，选择旧版模式后完整显示所有节点、连线和房间类型。
- 背包拖动改为事务操作：按下后从事务占格矩阵中暂时移除，松手前不修改正式背包数据。
- 拖动目标按玩家抓住的物品格对齐，不再按幽灵元素的像素偏移四舍五入。
- 空位移动和单物品双向可容纳交换统一使用占格矩阵验证；覆盖多个物品或越界时禁止提交。
- 可放置格显示绿色，冲突或越界显示红色；非法松手播放返回原位动画。
- 武器栏装卸改为先完整验证再一次提交，失败时背包数据与位置保持不变。
- 非正方形物品支持旋转：拖动中按 `R` 可在事务预览内旋转，点击物品也可使用“旋转”操作。
- 新增“一键整理”：按面积、长边、短边和物品 ID 固定排序，使用固定方向与行优先顺序进行确定性装箱。
- 旋转拖放和一键整理都先完成完整布局验证，再一次性提交；失败时背包位置与物品尺寸保持不变。
- 满包拾取改为“待处理物品”流程：自动放入失败后暂停游戏，打开左侧背包、右侧新物品和底部丢弃暂存区。
- 玩家可整理、旋转、暂存旧物品，再把新物品拖入背包；只有新物品已经入包时才能确认拾取。
- 暂存物品在确认拾取后才永久删除；“撤销丢弃”恢复最近一次暂存前的完整背包状态。
- `Esc` 和背包快捷键不能关闭待处理界面，必须明确确认拾取或放弃新物品。
- 幸运星和贪婪诅咒的额外物品统一进入同一拾取队列，不再因满包直接丢失。
- 收藏品图鉴只在成功撤离时登记，死亡或放弃的新物品不会提前解锁收藏记录。
- 待处理物品界面在竖屏舞台内整体居中，小屏时双栏自适应缩放，不再贴住画幅顶部。
- 局外资源拆分为金库金币与遗迹印记；首次带出金/红收藏品、首次击败章节 BOSS 各获得遗迹印记。
- 背包扩展为 `6×5 → 6×6 → 7×6 → 7×7 → 8×7`，升级同时校验撤离/章节条件、金币和印记。
- 背包格子根据当前行列和界面可用空间动态缩放，拖动、旋转、红绿预览与一键整理共用动态格子尺寸。
- 生命/神志训练上限提升至 10 级，武器校准 8 级，换弹与护盾扩容 5 级；章节前分别封顶 3/6/全部等级。
- 普通敌人生命改为“种类基础生命 × 章节倍率 × 本章深度倍率”，章节倍率为 `1.0 / 1.45 / 2.05`，章内每层 `+2.5%`。
- 第二章敌人数量按章内深度缓慢增长，主要通过远程、重甲、精神敌人的组合提高难度。
- 局外普通收藏图鉴只展示现有 6 件金色、5 件红色和新增 9 件章节特殊收藏，共 20 件；白、绿、蓝、紫不再登记或显示。
- 三件 BOSS 遗物移入独立“章节遗物”分页，不占普通收藏的 20 件进度。
- 每章新增 2 件特殊金色和 1 件特殊红色收藏，分别绑定宝藏/储物、诅咒/精英、深层宝藏/无伤清房等章节来源。
- 特殊收藏使用独立刷新池：金色基础概率约 `0.3%-0.8%`，红色约 `0.03%-0.12%`，不会进入普通宝物权重表。
- 有效箱子搜索连续 20 次未出现特殊金色后，金色概率开始逐次增加；特殊金色出现后保底计数归零，并跨局保存。
- 特殊收藏仍必须成功撤离才解锁图鉴和首次印记，拾取后放弃、死亡或未带出均不登记。
- 每个箱子会生成 2-3 件物品，因此可以连续搜索 2-3 次；搜空后才会失去高亮并变为灰色。
- 未搜空箱子的场景高亮统一为白色，搜索进度环仍按当前物品品质显示白、绿、蓝、紫、金/红等颜色。
- 补给房和宝藏房生成 2-4 个箱子；新增储物间，固定生成 2-3 个箱子，无怪物且进入不扣理智。
- 商店和篝火房与起点的最短图距离至少为 4 步，不会出现在起点 3 步以内。
- 起点现在对应一个可实际进入的入口回廊，仅保留通往相邻节点的路线门，不生成箱子、拾取物、火把或敌人。
- 起点固定在左下，撤离点固定在右下，BOSS 房固定在右上。
- 其余 18 个节点每局从带安全间距的二维候选点中随机抽取，并以双向边连接；地图包含多个环路，玩家可以回头或绕圈。
- 所有节点（含固定关键点）都参与最小距离检查，避免圆形节点互相遮挡。
- 每条连线生成前都会检查与第三方节点的距离，禁止路线穿过或擦过其他节点圆圈。
- 每个节点最多连接 3 个相邻房间，对应物理空间中的左、前、右三扇路线门。
- 出口门位按相对位置统一分配：1 个出口居中；2 个出口为左中或中右；3 个出口固定左中右，地图卡片与物理门保持一致。
- 房间会记录上一个节点作为进入方向；原路返回时会重新计算相对门位，并镜像、重排箱子与散落装饰。
- 已经走过的目标房间在选路卡片上显示“已搜刮”，不再重复显示左侧、前方或右侧。
- 路线选择中的当前目标使用闪烁黄圈，已探索节点保持静态黄圈，便于快速区分。
- 生成时只校验全图连通、环路数量、关键路径长度，以及起点到 BOSS/撤离点的最短路径重复节点不超过 3 个；不计算路线收益期望。
- 已探索节点显示黄色圆环。返回房间不重复扣理智、不重新刷怪。
- 地图默认完整全览；鼠标滚轮可在 `1x-2.8x` 间缩放，按住左键拖动平移，缩放中心跟随鼠标位置。
- 箱子、祭坛、篝火、商店和 BOSS 状态跟随地图节点保存；返回时不会凭空重置道具。
- BOSS 房和固定撤离点都可撤离：击败 BOSS 后可原地结算；固定撤离点抵达即可随时撤离，不要求击败 BOSS。
- 固定撤离点不会自动弹窗：进入房间后点击绿色撤离环，再选择“确认撤离”或“暂不撤离”；这里不提供继续深入选项。

依据策划案《古墓不让进2.0》开发的轻射击 PVE 搜打撤 RPG 原型。
TypeScript + Three.js，零 npm 依赖（Node 内置 TS 类型剥离构建）。

## 怎么玩

双击 `index.html`（需联网加载 Three.js CDN）。

- **自动前进**，遭遇敌人自动停下进入战斗（策划案4状态：移动/战斗/选择/搜索）
- **鼠标瞄准 + 左键射击**，爆头3倍伤害，弹夹12发，R/打空自动换弹
- **箱子搜刮**：每个箱子可搜索 2-3 次，读条颜色表示当前物品品质，箱子搜空后高亮消失
- **路线选择**：通过地图或房间内路线门前往相邻节点，每个节点最多连接 3 个房间
- **储物间**：无怪物、无理智消耗，直接搜索房内 2-3 个箱子
- **祭坛**：自动获得随机赐福（伤害/生命/神智）
- **神智**：每进新房扣神智，归零强制撤离（保留战利品）
- **撤离点**：点击绿色撤离环后选择确认或取消；撤离点不提供继续深入选项
- **死亡 = 丢失本局全部战利品**（搜打撤核心反馈）
- 微恐氛围：黑暗+手电筒+火把闪烁+低神智心跳与暗角

## 项目结构

```
src/
  input.ts    鼠标瞄准/点击/按键
  audio.ts    程序化音效（枪声为噪声合成）+ 暗黑环境音
  meta.ts     localStorage：金库/最深记录/撤离率
  loot.ts     战利品掉落表（深度加成）
  rooms.ts    房间生成器（战斗/储物/宝藏/祭坛/入口回廊/撤离点，动态拼接回收）
  enemy.ts    丧尸（头/身分部位hitbox、近战、倒地动画）
  player.ts   越肩相机+手电+枪械（弹夹/换弹/后座/枪口火光）
  hud.ts      血条/神智/弹药/准星/命中标记/搜索条/岔路与撤离UI
  game.ts     单局状态机与全部接线
  main.ts     开始界面+金库展示
build.js / index.template.html / index.html（双击即玩）
VERSION_PLAN.md  三版本拆分计划
archive/         每版本备份
```

## V2 新增
- 网格背包+品质（白/绿/蓝/紫/金/红），满包时由玩家整理、暂存丢弃或明确放弃新物品
- 消耗品独立槽：1医疗 2镇静 3手雷（准星处AoE爆炸）
- 敌人：开膛手(快)/追踪者(远程弹幕)/再生者(重甲，身体减伤必须打头)/无形之子(半透明，近战扣神智)/巨人(精英)
- 安全屋检查点（下局可从深处出发）、精英房战前情报

## V3 新增
- 三章节主题：金字塔/玛雅神庙/秦陵地宫（30层一章，材质配色与剧情引导切换）
- 章节BOSS（沙之暴君/羽蛇祭司/青铜俑将）：半血暴怒，冲锋+腐蚀弹+召唤，掉章节圣物
- 军械库与养成（金库消费）：霰弹枪、弹夹、助手，以及章节限制的生命/神志/武器校准/换弹/护盾成长
- 助手·夜枭：被动搜索+35%/每房回神智，主动群体电击（4键，每局3次）
- 收藏品图鉴：13种藏品收集进度（跨局持久化）

## V6 第三人称射击体验
- **第三人称全身视角**：角色全身可见（探险者模型：束发/背包/双枪臂），相机后上方纵览，角色仅占画面下部小幅空间（参考竖屏 TPS 构图）
- **自动瞄准/射击**：进入战斗房后自动转向并瞄准最近敌人「身体」持续开火——身体命中**不触发暴击**（稳定输出）
- **手动 ADS**：按住屏幕/左键进入瞄准——相机 FOV 收紧（68°→44°）、画面中心出现准星、子弹落点为摄像机中心（瞄头可暴击）；拖动控制射击方向
- 枪口曳光从角色枪口射出；松开即回到第三人称自动模式

## V5 深入契约（肉鸽赌注）
- **撤离点抉择强化**：选择"继续深入"时，弹出 2 选 1（可婉拒）的「深入契约」——每张契约提供一项强力增益，并附带一项永久代价，可跨多次深入叠加，越深越强也越脆
  - 🩸 血祭契约：武器伤害 +30% ｜ 代价 最大生命 -25
  - 👑 贪婪契约：撤离战利品价值 +35%（结算时生效）｜ 代价 神智上限 -18
  - ⚡ 亡命契约：前进/搜索速度 +35% 并赠 1 手雷 ｜ 代价 受到伤害 +20%
- 结算界面显示契约价值加成的换算明细

## V4 体验优化
- **竖屏适配**：9:16舞台居中渲染，宽屏两侧自动露出程序生成的砂岩象形纹（荷鲁斯之眼/安卡/水波/金字塔/圣甲虫浮雕贴片）；触屏按住即瞄准开火，可直接手机玩
- **石门仪式感**：每间房出口换成雕花双开石门——鎏金描边+符文浮雕门板、门柱+鎏金门楣+圣甲虫徽记，走近时石门伴随低频隆隆声缓缓滑入墙内
- **场景提亮**：环境光+半球补光增强、雾距13→46、每房火把2→4支且亮度提升
